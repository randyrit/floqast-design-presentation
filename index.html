05 — The Real Problem

One Component, Twelve Versions

Which dropdown do you use?

A single search for "dropdown" in the codebase returned over a dozen variants — each built by a different team, at a different time, with different APIs. No naming convention, no shared contract, no single source of truth.

12+
dropdown variants across the codebase — designers couldn't tell which was current, engineers couldn't tell which was maintained

Some were versioned (DropdownV2, DropdownV3), some were team-specific (dropdown_nav), and one was literally called DROPDOWN_BACKUP. Every new hire picked a different one. Every product surface looked slightly different.

11 — Typography Rationale

Why Inter + Museo Sans?

Museo Sans
Headings & Display
psychology

Warm but authoritative. Museo's soft, rounded terminals give enterprise UI a human touch without sacrificing professionalism — accountants trust it, designers love it.

visibility

Instant hierarchy. The weight contrast between Museo headings and Inter body creates clear visual separation — users can scan dense financial data without friction.

fingerprint

Brand distinction. In a sea of enterprise products using the same geometric sans, Museo gives FloQast a recognizable voice without resorting to novelty.

Inter
Body & UI
desktop_windows

Born for screens. Inter was designed specifically for computer interfaces. Tall x-height, open apertures, and tabular figures — perfect for data-heavy accounting workflows.

text_fields

Legible at any size. From 10px table cells to 16px body copy, Inter stays crisp. Disambiguation between Il1 and O0 matters when you're reading financial figures all day.

speed

Variable font, zero compromise. One file, infinite weights. We load a single variable font instead of 6 static files — faster pages, consistent rendering across every platform.

16 — Transformation

Before & After Flow-UI

Before

  • Every team built their own button, modal, and form components
  • No shared design tokens — hard-coded hex values everywhere
  • WCAG failures on 60%+ of audited pages
  • Design-to-dev handoff took days of back-and-forth
→

After

  • 40+ shared components with variants, auto layout, and Storybook parity
  • Semantic design tokens synced between Figma and code
  • WCAG AA compliance across all product surfaces
  • Figma-to-code in hours, not days — with Code Connect
